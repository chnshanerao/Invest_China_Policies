# 中国国家政策投资追踪系统

> Policy Investment Tracker — 追踪 92 项国家级战略政策的生命周期、执行力度、投资机会

## 系统概览

本系统对中国现行 92 项国家级战略政策进行全生命周期追踪，涵盖 10 大政策类别、72 个投资机会，并提供可视化仪表板、筛选分析和评分方法论文档。

**核心特性：**

- **7 阶段生命周期模型**：规划期 → 启动期 → 扩张期 → 验证期 → 成熟期 → 调整期 → 衰退期
- **量化评分体系**：执行力度 (1-10) 与执行效果 (1-10)，附评分依据与数据来源
- **政策动量评估**：加速 / 平稳 / 减速，基于近 1-2 年趋势
- **数据完整性保障**：41 项重点政策标注官方数据来源与政策文件链接，无法验证的字段一律置空
- **投资机会分析**：72 个投资机会，含行业、风险等级、投资期限、相关股票/ETF

## 系统架构

```
policy-tracker/
├── backend/
│   ├── app.py                  # FastAPI 主应用 (REST API + 静态文件)
│   ├── database.py             # SQLite 数据库初始化 (7 张表)
│   ├── seed_data.py            # 92 项政策 + 72 个投资机会种子数据
│   ├── data_collector.py       # 异步新闻采集 + 情绪分析
│   ├── data_sources.py         # 41 项政策的可验证数据来源
│   └── lifecycle_assessment.py # 生命周期评估 (阶段/力度/效果/动量)

│   ├── index.html              # 单页应用 (6 个标签页)
│   ├── css/style.css           # 暗色主题样式
│   └── js/app.js               # 前端交互与可视化
└── data/
    └── policies.db             # SQLite 数据库 (自动生成)
```

## 功能页面

### 1. 仪表板
生命周期分布、政策动量趋势、风险矩阵、类别概览、热门投资行业 TOP 12

### 2. 生命周期视图
政策卡片式布局，按生命周期阶段排列，支持按阶段/动量/类别筛选，可视化力度与效果评分

### 3. 明细数据
每个政策一行的全量数据表，17 列含：
- 基础信息：名称、类别、设立年份
- 生命周期：阶段、动量、力度评分、效果评分
- 财务数据：GDP、投资额
- **数据溯源**：政策文件链接、力度评分依据、效果评分依据、生命周期评估依据
- **数据验证状态**：未验证 / 部分验证 / 已验证

附"评分方法论"面板，详述评分标准、权重、数据来源

### 4. 政策列表
可筛选的政策卡片，支持按类别/风险/阶段/动量/关键词搜索

### 5. 投资机会
72 个投资机会，含行业、风险等级、预期回报、投资期限、相关股票代码和 ETF

### 6. 政策新闻
自动采集的政策相关新闻，附情绪分析（积极/中性/消极）

## 评分方法论

### 执行力度 (1-10)

| 维度 | 权重 | 数据来源 |
|------|------|----------|
| 资金投入规模 | 30% | 财政部预算报告、发改委批复 |
| 政策文件密度 | 20% | gov.cn 政策库 |
| 领导层关注度 | 25% | 政府工作报告、中央经济工作会议 |
| 配套措施完备度 | 25% | 全国人大法律库、部委规章 |

### 执行效果 (1-10)

| 维度 | 权重 | 数据来源 |
|------|------|----------|
| 经济指标达成 | 30% | 国家统计局、各省市统计公报 |
| 就业与人口效应 | 15% | 人社部、统计局人口普查 |
| 产业集聚效应 | 25% | 工信部、各产业协会 |
| 制度创新贡献 | 15% | 商务部自贸区经验推广通报 |
| 社会/生态效益 | 15% | 生态环境部、卫健委 |

### 数据完整性规则

- 所有 GDP、投资、财政等数值必须有官方统计来源
- 无法验证的字段标记为 NULL（界面显示"-"）
- 评分依据必须注明具体数据来源
- `data_verified`: 0=未验证 / 1=部分验证 / 2=已验证

## 快速启动

```bash
# 安装依赖
pip install fastapi uvicorn aiohttp beautifulsoup4

# 启动服务 (自动初始化数据库和种子数据)
cd policy-tracker
python3 backend/app.py

# 访问 http://localhost:8080
```

## API 接口

| 接口 | 说明 |
|------|------|
| `GET /api/dashboard/summary` | 仪表板汇总数据 |
| `GET /api/policies` | 政策列表（支持筛选和排序） |
| `GET /api/policies/{id}` | 单个政策详情 |
| `GET /api/policies/detail-table` | 明细数据表 |
| `GET /api/opportunities` | 投资机会列表 |
| `GET /api/news` | 政策新闻 |
| `GET /api/filters` | 筛选选项 |
| `GET /api/scoring-methodology` | 评分方法论 |
| `GET /api/categories` | 政策类别 |
| `GET /api/sectors` | 行业分布 |
| `POST /api/update/trigger` | 手动触发数据更新 |

## 技术栈

- **后端**：Python 3 / FastAPI / SQLite (WAL mode)
- **前端**：原生 HTML / CSS / JavaScript（无框架依赖）
- **数据采集**：aiohttp + BeautifulSoup（百度新闻）
- **定时更新**：asyncio 周任务（每周自动采集新闻）
